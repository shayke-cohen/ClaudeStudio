# Performance Round 2 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Apply five targeted performance improvements identified in a second-pass audit: fix O(N×M) sidebar expand, remove a dead full-table query, migrate message list to virtualized List, and migrate AppState from ObservableObject (full re-render on every token) to @Observable (property-level tracking).

**Architecture:** All changes are purely additive or refactors with no user-visible behavior change. The @Observable migration is the highest-risk item and is sequenced last so earlier, lower-risk wins land first. Each task ends with `make build-check` to catch regressions immediately.

**Tech Stack:** Swift 6, SwiftUI, SwiftData, macOS 14+, @Observable macro (Observation framework, no Combine required)

## Task 1: Fix O(N×M) expandForReveal + remove dead allSessions query

**Files:**
- Modify: `Odyssey/Views/MainWindow/SidebarView.swift:1919-1938`
- Modify: `Odyssey/Views/MainWindow/ChatView.swift:254,285-291`

### Context

`expandForReveal()` is called when the sidebar needs to auto-expand to reveal a conversation (e.g. when navigating to a conversation from a notification). It currently iterates every agent and for each agent calls `conversationsForAgent(agent)` which traverses `agent.sessions` lazy relationship — O(N agents × M sessions) fetches every call.

`allSessions` is a full `@Query` (every session in the DB) used only as a fallback in `conversationSessions` at line 288. The relationship (`conversation?.sessions`) is always populated when a conversation is loaded, making this fallback dead code.

### Steps

- [ ] **Step 1.1: Add the reverse-index state and a rebuild helper to SidebarView**

In `SidebarView.swift`, add a `@State` dictionary just below the existing `@State private var cachedSortedProjects` block, and a helper to rebuild it:

```swift
// After the existing @State caches (around line 190):
@State private var conversationToAgentIndex: [UUID: UUID] = [:]

private func rebuildConversationIndex() {
    var index: [UUID: UUID] = [:]
    for agent in agents {
        for session in agent.sessions {
            for convo in session.conversations {
                index[convo.id] = agent.id
            }
        }
    }
    conversationToAgentIndex = index
}
```

- [ ] **Step 1.2: Wire the rebuild to data-change events**

In the `body` of `SidebarView`, alongside the existing `.onAppear` / `.onChange` cache rebuilds, add:

```swift

    rebuildConversationIndex()
}
.onChange(of: conversations.count) { _, _ in rebuildConversationIndex() }
.onChange(of: agents.count) { _, _ in rebuildConversationIndex() }
```

- [ ] **Step 1.3: Replace expandForReveal() implementation**

Replace the existing body of `expandForReveal()` at `SidebarView.swift:1919-1938` with the O(1) index lookup:

```swift
private func expandForReveal(_ conversationId: UUID) {
    guard let convo = conversations.first(where: { $0.id == conversationId }) else { return }
    let isArchived = convo.isArchived

    if let groupId = convo.sourceGroupId {
        expandedGroupIds.insert(groupId)
        if isArchived { expandedArchivedGroupIds.insert(groupId) }
        return
    }

    if let agentId = conversationToAgentIndex[conversationId] {
        expandedAgentIds.insert(agentId)
        if isArchived { expandedArchivedAgentIds.insert(agentId) }
    }
}
```

## Task 2: Migrate message list from LazyVStack to List

**Files:**
- Modify: `Odyssey/Views/MainWindow/ChatView.swift:1608-1700` (messageList view builder)

### Context

`messageList` is a `ScrollViewReader { ScrollView { LazyVStack { ForEach(displayMessages) { ... } } } }`. `LazyVStack` is lazy in creation (defers body computation until a row is near the visible area) but **never recycles** — once a `MessageBubble` is created, it stays alive for the lifetime of the conversation. For long conversations (200+ messages) this means hundreds of alive `MessageBubble` trees, each with their own `MarkdownContent`, `CodeBlockView`, and attachment views.

`List` on macOS uses `NSTableView` under the hood which virtualizes rows — off-screen rows are deallocated. It supports `ScrollViewReader` wrapping for programmatic scrolling and `.id()` for row targeting. The `GeometryReader` preference-key approach for scroll-position tracking still works inside `List` rows.

Visual parity requires:
- `.listStyle(.plain)` — removes grouped background
- `.listRowSeparator(.hidden)` — removes dividers between messages
- `.listRowBackground(Color.clear)` — transparent row background
- `.listRowInsets(EdgeInsets(top: 6, leading: 0, bottom: 6, trailing: 0))` — matches the `spacing: 12` of LazyVStack

### Steps

- [ ] **Step 2.1: Replace ScrollView + LazyVStack with List**

In `messageList`, the structure starting at line 1608:


## Task 3: Migrate AppState from ObservableObject to @Observable

**Files:**
- Modify: `Odyssey/App/AppState.swift`
- Modify: 40 view files (all `@EnvironmentObject` → `@Environment`, all `.environmentObject` → `.environment`)
- Modify: `Odyssey/Views/MainWindow/ChatView.swift` (4 `.onReceive` → `.onChange`)

### Context

`AppState` is `final class AppState: ObservableObject` with 37 `@Published` properties. Any write to any `@Published` property calls `objectWillChange.send()`, which triggers a re-render on **all** 40+ views that have `@EnvironmentObject var appState: AppState`. During streaming, `streamingText[sessionId]` is updated on every token — causing ~43 views to re-render per token, most of which don't read `streamingText`.

Migrating to `@Observable` (Swift 5.9 / macOS 14) makes SwiftUI track which specific properties each view accesses at render time. Only views that read a changed property re-render. A `DebugLogView` that only reads `appState.sidecarStatus` will not re-render on every streaming token.

**Breaking API change:**
- `@Published` is removed — Combine publishers like `appState.$streamingText` no longer exist
- `.onReceive(appState.$xxx)` patterns must become `.onChange(of: appState.xxx)`
- `@EnvironmentObject` → `@Environment` (different injection/reading API)

### Steps

- [ ] **Step 3.1: Update AppState.swift**

1. Remove `import Combine` (line 3).
2. Replace `final class AppState: ObservableObject {` with `@Observable @MainActor final class AppState {`.
3. Strip all `@Published` and `@Published private(set)` annotations from every property (keep `private(set)` where it was `@Published private(set)`).

## Risk Notes

**Task 2 (LazyVStack → List):** If the `GeometryReader` preference-key approach for the scroll-to-bottom button breaks (the button appears/disappears incorrectly), revert this task — the O(1) expand and @Observable migration are independent wins.

**Task 3 (@Observable):** If there are `@Published` properties used outside SwiftUI views (e.g. Combine sinks in other `ObservableObject` types that subscribe to AppState), those will break. Check for any `.sink(receiveValue:)` or `.assign(to:)` calls on AppState properties before committing.
